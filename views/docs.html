API Sistema de Reservas de Salas

Esta aplicação expõe três conjuntos de endpoints REST em /api/usuarios, /api/salas e /api/reservas. Todos aceitam e devolvem JSON.

1. Usuários

  • GET /api/usuarios – lista todos os usuários.
  • POST /api/usuarios – cria usuário.
    Exemplo de corpo: {"nome":"Ana Silva","email":"ana@exemplo.com"}
  • DELETE /api/usuarios/:id – remove usuário.

2. Salas

  • GET /api/salas – lista todas as salas.
  • POST /api/salas – cria sala.
    Exemplo: {"nome":"Sala de Reunião A","capacidade":10,"disponivel":true}
  • GET /api/salas/:id – detalhe da sala.
  • PUT /api/salas/:id – atualiza sala.
  • DELETE /api/salas/:id – remove sala.

3. Reservas

  • GET /api/reservas – lista todas as reservas.
  • POST /api/reservas – cria reserva.
    Exemplo: {"idUsuario":1,"idSala":2,"dataInicio":"2024-12-01T09:00:00","dataFim":"2024-12-01T11:00:00"}
  • GET /api/reservas/:id – detalhe da reserva.
  • PUT /api/reservas/:id – atualiza reserva.
  • DELETE /api/reservas/:id – remove reserva.

Exemplos de uso via curl

# Criar usuário
curl -X POST http://localhost:3000/api/usuarios \
     -H "Content-Type: application/json" \
     -d '{"nome":"Ana Silva","email":"ana@exemplo.com"}'

# Criar sala
curl -X POST http://localhost:3000/api/salas \
     -H "Content-Type: application/json" \
     -d '{"nome":"Sala de Reunião A","capacidade":10,"disponivel":true}'

# Criar reserva
curl -X POST http://localhost:3000/api/reservas \
     -H "Content-Type: application/json" \
     -d '{"idUsuario":1,"idSala":1,"dataInicio":"2024-12-01T09:00:00","dataFim":"2024-12-01T11:00:00"}'

# Listar salas disponíveis
curl http://localhost:3000/api/salas-disponiveis

# Listar reservas detalhadas
curl http://localhost:3000/api/reservas-detalhadas
